TROUBLESHOOTING:
═══════════════════════════════════════════════════════════════════════════

Problem 1: "Invalid access code" error
───────────────────────────────────────
Cause: Code doesn't exist in database
Solution:
  1. Check Supabase → access_codes table
  2. Verify code is exactly 8 characters
  3. Make sure is_active = true
  4. Try a different code
  5. Clear browser cache (Ctrl+Shift+Delete)

Problem 2: Login button not working
───────────────────────────────────
Cause: Environment variables not set
Solution:
  1. Check NEXT_PUBLIC_SUPABASE_URL env var
  2. Check NEXT_PUBLIC_SUPABASE_ANON_KEY env var
  3. Restart the server: npm run dev
  4. Check browser console (F12) for errors

Problem 3: Can't reach http://localhost:3000
──────────────────────────────────────────────
Cause: Server not running
Solution:
  1. Run: npm run dev
  2. Wait for "ready - started server on 0.0.0.0:3000"
  3. Try http://localhost:3000 again

Problem 4: Dashboard shows no data
──────────────────────────────────
Cause: No sales/expense records yet
Solution:
  1. Add some sales records first
  2. Add some expenses
  3. Refresh the page
  4. Charts should now show data

Problem 5: "Cannot find module" error
─────────────────────────────────────
Cause: Dependencies not installed
Solution:
  1. Run: npm install
  2. Wait for it to complete
  3. Run: npm run dev

Problem 6: Database connection error
───────────────────────────────────
Cause: Supabase not connected
Solution:
  1. Check Supabase status: https://status.supabase.com
  2. Verify env variables are set
  3. Check internet connection
  4. Try logging in again
